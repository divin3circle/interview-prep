# Evaluation Criteria Breakdown

## Comprehensive Interview Evaluation Framework

Technical interviews assess candidates across multiple dimensions to determine their suitability for the role. This document provides a detailed breakdown of the evaluation criteria used by interviewers.

## Primary Evaluation Dimensions

### 1. Technical Skills (30% weight)

**Algorithm Knowledge**:
- Understanding of fundamental algorithms (sorting, searching, traversal)
- Ability to apply algorithms to novel problems
- Knowledge of algorithm complexity and optimization techniques
- Familiarity with advanced algorithms for senior roles

**Data Structure Mastery**:
- Deep understanding of core data structures (arrays, linked lists, trees, graphs, hash tables)
- Ability to choose appropriate data structures for specific problems
- Knowledge of data structure operations and their complexities
- Understanding of tradeoffs between different data structures

**Computer Science Fundamentals**:
- Big-O notation and complexity analysis
- Recursion and iteration
- Memory management concepts
- Basic system design principles (for mid-level and above)

**Evaluation Criteria**:
- Can the candidate recall and apply fundamental concepts?
- Do they understand the theoretical foundations?
- Can they discuss complexity and performance characteristics?

---

### 2. Problem Solving (35% weight)

**Approach and Methodology**:
- Systematic problem decomposition
- Clarification of requirements and constraints
- Identification of edge cases
- Structured thinking process

**Pattern Recognition**:
- Ability to recognize problem patterns
- Connection to previously solved problems
- Application of known techniques to new contexts
- Creative adaptation of standard approaches

**Optimization Ability**:
- Starting with brute force and iterating to optimal
- Identifying bottlenecks in solutions
- Understanding time-space tradeoffs
- Recognizing when solution is optimal

**Debugging and Testing**:
- Ability to identify bugs in own code
- Systematic debugging approach
- Consideration of test cases and edge conditions
- Verification of solution correctness

**Evaluation Criteria**:
- Does the candidate have a systematic approach?
- Can they optimize solutions effectively?
- Do they think critically about their solution?
- Can they identify and fix issues independently?

---

### 3. Code Quality (20% weight)

**Structure and Organization**:
- Logical code organization
- Appropriate use of functions and modularity
- Clear separation of concerns
- Maintainable code structure

**Readability**:
- Meaningful variable and function names
- Consistent naming conventions
- Appropriate use of comments
- Clean formatting and indentation

**Correctness**:
- Code compiles and runs without errors
- Handles all specified cases correctly
- Properly manages edge cases
- No off-by-one errors or boundary issues

**Best Practices**:
- Follows language idioms and conventions
- Appropriate error handling
- Efficient use of language features
- Professional coding standards

**Evaluation Criteria**:
- Is the code production-quality?
- Would this code pass code review?
- Is it maintainable by other engineers?
- Does it follow best practices?

---

### 4. Communication (15% weight)

**Verbal Explanation**:
- Clear articulation of thought process
- Ability to explain complex concepts simply
- Logical flow in explanations
- Appropriate technical terminology

**Collaboration**:
- Receptiveness to hints and suggestions
- Professional response to feedback
- Asking clarifying questions
- Working effectively with interviewer

**Presentation**:
- Organized explanation of solution
- Clear complexity analysis discussion
- Effective use of examples
- Confidence without arrogance

**Active Thinking**:
- Thinking aloud during problem-solving
- Explaining decisions and tradeoffs
- Discussing alternative approaches
- Transparent reasoning process

**Evaluation Criteria**:
- Can the candidate explain their thinking clearly?
- Do they collaborate effectively?
- Would they communicate well on a team?
- Can they articulate technical concepts?

## Secondary Evaluation Factors

### Cultural Fit and Soft Skills

**Professionalism**:
- Appropriate demeanor and attitude
- Respectful interaction
- Time management
- Handling of stress

**Learning Ability**:
- Response to new information
- Adaptation when stuck
- Incorporation of feedback
- Growth mindset indicators

**Enthusiasm and Interest**:
- Genuine interest in the role and company
- Thoughtful questions about the team and work
- Engagement during the interview
- Passion for technology and problem-solving

### Level-Appropriate Expectations

**Junior/Entry-Level**:
- Strong fundamentals expected
- May need hints for optimization
- Focus on learning ability and potential
- Communication and collaboration critical

**Mid-Level**:
- Independent problem-solving expected
- Should reach optimal solutions with minimal hints
- Clean code and best practices required
- Strong communication essential

**Senior-Level**:
- Optimal solutions expected quickly
- System design and architecture thinking
- Mentorship and leadership indicators
- Strategic thinking about tradeoffs

## Scoring and Decision Making

### Signal Strength Classification

**Strong Positive Signal**:
- Exceeds expectations for level
- Demonstrates mastery in multiple areas
- Minimal interviewer assistance needed
- Clear hire recommendation

**Positive Signal**:
- Meets expectations for level
- Solid performance across dimensions
- Some assistance acceptable
- Likely hire recommendation

**Neutral Signal**:
- Borderline performance
- Strengths and weaknesses balance out
- Significant assistance required
- Uncertain recommendation

**Negative Signal**:
- Below expectations for level
- Struggles in multiple areas
- Cannot complete problem even with help
- No hire recommendation

### Aggregation Across Rounds

**Hiring Decision Factors**:
- Consistency across multiple interview rounds
- No major red flags in any dimension
- Strong signals in critical areas for the role
- Team consensus on candidate fit

**Common Rejection Reasons**:
- Weak signal in any critical dimension
- Inconsistent performance across rounds
- Poor communication despite technical ability
- Inability to handle feedback or hints
- Lack of fundamental knowledge

## Maximizing Performance Across All Dimensions

### Technical Skills
- Study fundamental algorithms and data structures thoroughly
- Practice complexity analysis on all problems
- Understand when and why to use each data structure
- Review computer science fundamentals regularly

### Problem Solving
- Develop and practice a systematic framework
- Build pattern recognition through deliberate practice
- Always start with brute force, then optimize
- Practice identifying edge cases systematically

### Code Quality
- Write production-quality code even under time pressure
- Use meaningful names consistently
- Structure code logically from the start
- Test mentally before declaring completion

### Communication
- Practice thinking aloud while solving problems
- Explain your approach before coding
- Respond positively to all feedback
- Ask clarifying questions proactively
