# Contributing to the Interview Prep Course

Thank you for your interest in contributing! This repository aims to be the gold standard for technical interview preparation. We maintain high standards for technical accuracy, code quality, and professional explanations.

## Code of Conduct
- Be respectful and constructive in pull requests and issues.
- Provide feedback to help others learn.

## How to Contribute

### 1. Reporting Issues
- Check the issue tracker to see if the issue has already been reported.
- Open a new issue with a clear title and description.
- Include reproduction steps for code issues.

### 2. Submitting Solutions
- **Language**: Java is the primary language.
- **Format**: Follow the `resources/templates/solution-template.java`.
- **Quality**:
  - Variable names should be descriptive (e.g., `fastPointer` instead of `f`).
  - Comments should explain *why*, not just *what*.
  - Include Time and Space complexity analysis.

### 3. Improving Documentation
- Explanations should be professional and technical.
- **No emojis** in official documentation files.
- Use clear markdown formatting (headers, code blocks).

## Pull Request Process
1. Fork the repository.
2. Create a feature branch (`git checkout -b feature/amazing-feature`).
3. Commit your changes (`git commit -m 'Add amazing feature'`).
4. Push to the branch (`git push origin feature/amazing-feature`).
5. Open a Pull Request.

## Directory Structure
- `phase-1-dsa-foundations`: Core data structures and algorithms.
- `phase-2-core-patterns-blind75`: The Blind 75 problem set organized by pattern.
- `resources`: Cheatsheets, templates, and study plans.

We look forward to your contributions!
