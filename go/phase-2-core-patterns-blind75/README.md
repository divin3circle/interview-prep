# Phase 2: Core Patterns + Blind 75

## Overview

Phase 2 acts as the core of this interview prep course. It covers the **Blind 75** curated problem set, adapted for Go. These problems teach essential algorithmic patterns that appear most frequently in technical interviews.

**Duration**: Variable (Recommend 8-12 weeks)
**Modules**: 12
**Total Problems**: 45+
**Prerequisites**: Phase 0 and Phase 1 completion

## Learning Objectives

By completing Phase 2, you will:
- Master 12 fundamental algorithmic patterns in Go.
- Solve all Blind 75 problems with optimal Go solutions.
- Recognize problem patterns instantly.
- Apply Go-specific optimizations (e.g., slices, maps, `strings.Builder`).
- Analyze Time/Space complexity.

## Why Blind 75?

The Blind 75 is a curated list of LeetCode problems that:
- Covers all fundamental patterns.
- Balances breadth and depth.
- Is highly representative of top-tier interview questions.

## Module Structure

Each module includes:
1. **Pattern Introduction**: Explanation of the pattern.
2. **Go-Specific Tips**: How to implement the pattern idiomatically in Go.
3. **Problems**: 2-6 problems demonstrating the pattern.

## Modules Overview

### Module 3: Arrays & Hashing
**Pattern**: Hash Maps (`map[K]V`) and Sets (`map[T]bool`) for O(1) lookups.
- Contains Duplicate
- Two Sum
- Valid Anagram
- Group Anagrams
- Top K Frequent Elements
- Product of Array Except Self

### Module 4: Two Pointers
**Pattern**: Two pointers traversing arrays/strings efficiently.
- Valid Palindrome
- Two Sum II
- 3Sum
- Container With Most Water
- Trapping Rain Water

### Module 5: Sliding Window
**Pattern**: Maintaining a window of elements.
- Best Time to Buy and Sell Stock
- Longest Substring Without Repeating Characters
- Longest Repeating Character Replacement
- Minimum Window Substring

### Module 6: Stack
**Pattern**: LIFO for nested structures/monotonic sequences.
- Valid Parentheses
- Min Stack
- Evaluate Reverse Polish Notation
- Daily Temperatures
- Car Fleet
- Largest Rectangle in Histogram

### Module 7: Binary Search
**Pattern**: Searching sorted spaces in O(log n).
- Binary Search
- Search in Rotated Sorted Array
- Find Minimum in Rotated Sorted Array
- Search a 2D Matrix

### Module 8: Linked Lists
**Pattern**: Pointer manipulation.
- Reverse Linked List
- Merge Two Sorted Lists
- Reorder List
- Remove Nth Node From End
- Linked List Cycle
- Add Two Numbers

### Module 9: Trees (DFS & BFS)
**Pattern**: Recursion and Queue-based traversal.
- Max Depth
- Same Tree
- Subtree of Another Tree
- Invert Binary Tree
- LCA of BST
- Level Order Traversal

### Module 10: BST
**Pattern**: BST properties.
- Validate BST
- Kth Smallest Element

### Module 11: Heaps
**Pattern**: Priority Queue (`container/heap`) for Top-K.
- Kth Largest Element
- Find Median from Data Stream

### Module 12: Backtracking
**Pattern**: Exhaustive search with pruning.
- Subsets
- Permutations
- Combination Sum
- Word Search

### Module 13: Graphs
**Pattern**: Connectivity (DFS/BFS, Union Find).
- Number of Islands
- Clone Graph
- Pacific Atlantic Water Flow
- Course Schedule

### Module 14: Dynamic Programming
**Pattern**: Overlapping subproblems.
- Climbing Stairs
- House Robber
- Coin Change
- LIS
- Word Break
- Unique Paths

## Study Approach

1. **Go Idioms**: Focus on writing clean Go code (e.g., avoiding global state, handling errors/edge cases gracefully).
2. **Framework**: Use the 10-step framework for every problem.
3. **Tests**: Use `go test` to verify your solutions.
